# Thor Video Player

Лендінг сторінка у стилі TikTok для демонстрації ігрових відео, побудована на Vue 3 + Vite.

## 🚀 Швидкий старт

### Встановлення залежностей
```bash
npm install
```

### Розробка
```bash
npm run dev
```

### Збірка для продакшену
```bash
npm run build
```

### Попередній перегляд збірки
```bash
npm run preview
```

## 📦 Деплой на Cloudflare R2

Проєкт налаштований для автоматичного деплою на Cloudflare R2 бакет `winspirits-nl` у папку `trends-all-geo`.

### Команди деплою

```bash
# Повний деплой (збірка + завантаження)
npm run deploy
```

### Як працює деплой

1. **Збірка проєкту**: Виконується команда `npm run build`, яка створює оптимізовану версію у папці `dist/`

2. **Завантаження файлів**: Всі файли з папки `dist/` завантажуються на R2 бакет у папку `trends-all-geo/` зі збереженням структури директорій

3. **Підтримувані типи файлів**:
   - HTML файли (`.html`)
   - JavaScript файли (`.js`)
   - CSS файли (`.css`)
   - Зображення (`.webp`, `.svg`)
   - Відео (`.mp4`, `.webm`)
   - Конфігурація (`.json`)

4. **Структура завантаження**:
   ```
   winspirits-nl/
   └── trends-all-geo/
       ├── index.html
       ├── shorts/
       │   ├── index-[hash].css
       │   └── index-[hash].js
       ├── video/
       │   ├── h264/
       │   ├── webm/
       │   └── posters/
       └── [інші ресурси]
   ```

### Конфігурація деплою

Деплой використовує наступні налаштування:
- **Бакет**: `winspirits-nl`
- **Папка призначення**: `trends-all-geo/`
- **Аутентифікація**: API токен Cloudflare (вбудований у скрипт)
- **Регіон**: auto (автоматичний вибір)

### Технічні деталі

- **Скрипт деплою**: `scripts/deploy.sh`
- **Модуль завантаження**: `scripts/upload-direct.cjs`
- **Протокол**: HTTPS з AWS4-HMAC-SHA256 підписом
- **Ендпоінт**: `https://83802fb0b9b6916c3c04c4927f3c3010.r2.cloudflarestorage.com`

## 📹 Структура файлу масиву відео (current_videos.ts)

Файл `src/gameDataSets/current_videos.ts` містить масив `gamesData` з усіма ігровими відео та їх конфігурацією.

### Структура об'єкта гри (Game)

Кожна гра в масиві має наступні поля:

```typescript
{
  name: string;              // Назва гри (відображається в інтерфейсі)
  regions: {                 // Конфігурація для різних регіонів
    au?: {                   // Налаштування для Австралії
      game_slug: string;     // Унікальний ідентифікатор гри для AU
      provider: string;      // Назва провайдера для AU
      provider_slug: string; // Slug провайдера для AU
    };
    global?: {               // Налаштування для інших регіонів
      game_slug: string;     // Унікальний ідентифікатор гри для global
      provider: string;      // Назва провайдера для global
      provider_slug: string; // Slug провайдера для global
    };
  };
  src: string;               // Ім'я файлу відео (без розширення)
  supportedCurrencies: {     // Підтримувані валюти по регіонах
    au?: string[] | 'all';   // Валюти для AU ('all' або масив валют)
    global?: string[] | 'all'; // Валюти для global ('all' або масив валют)
  };
  no_demo?: boolean;         // Чи відключено демо-режим (для live ігор)
  external_link?: {          // Зовнішнє посилання (опціонально)
    url: string;             // URL адреса
    localeKey: string;       // Ключ локалізації для тексту кнопки
  };
  dont_hide?: boolean;       // Чи не приховувати гру після перегляду
}
```

### Опис полів

- **name**: Повна назва гри, яка відображається користувачу в інтерфейсі
- **regions**: Об'єкт з налаштуваннями для різних регіонів (AU та global). Дозволяє використовувати різні `game_slug` та провайдерів залежно від геолокації користувача
- **src**: Базове ім'я файлу відео. Використовується для пошуку відео у форматах `.mp4` та `.webm`, а також для постера `.webp`
- **supportedCurrencies**: Визначає які валюти підтримує гра у кожному регіоні. Значення `'all'` означає підтримку всіх валют, масив типу `['aud', 'usd']` обмежує список валют
- **no_demo**: Якщо `true`, кнопка "Грати демо" не відображається (використовується для live-ігор та інших ігор без демо-режиму)
- **external_link**: Опціональне поле для додавання зовнішнього посилання замість стандартної кнопки "Грати"
- **dont_hide**: Якщо `true`, гра залишається у списку навіть після того, як користувач переглянув 30% відео

### Приклад запису

```typescript
{
  name: 'Live Roulette French',
  regions: {
    au: {
      game_slug: 'live_roulette_french_vip',
      provider: 'Platipus Live',
      provider_slug: 'platipus_live',
    },
    global: {
      game_slug: 'live_roulette_french_vip',
      provider: 'Platipus Live',
      provider_slug: 'platipus_live',
    }
  },
  src: 'live_roulette_french_vip',
  supportedCurrencies: {
    au: 'all',
    global: 'all'
  },
  no_demo: true,  // Live гра без демо-режиму
}
```

### Функції-хелпери

Файл також експортує допоміжні функції:

- **isDemoDisabled()**: Перевіряє, чи відключено демо-режим для гри

## 🔄 Система версіонування відео та управління значком Trends

### Файл videos_version.json

Файл `public/videos_version.json` (доступний за URL `https://winspirits.nl/trends-all-geo/videos_version.json`) використовується для сповіщення користувачів сайту про наявність нових відео у рілсах.

```json
{
  "version": "17"
}
```

**Важливо**: Версія має бути цілим числом і збільшуватися по зростанню (17 → 18 → 19...).

### Логіка роботи системи версіонування

**Примітка**: Вся логіка перевірки версії та управління значком Trends реалізована на фронті основного сайту (головна сторінка та сторінка казино), а **не в самому плеєрі відео**. 

#### Що робить плеєр відео

Коли користувач переглядає всі відео у рілсах, плеєр відправляє івент `watched_all_video`. Цей івент обробляється на фронті сайту.

#### Що робить фронт сайту

На головній сторінці та сторінці казино реалізована наступна логіка:

**За замовчуванням**: На сторінці відображається неактивний thumbnail значка Trends.

**Асинхронна перевірка**: Після завантаження сторінки виконується асинхронний скрипт, який:

1. Перевіряє наявність запису у `localStorage` (створюється при отриманні івенту `watched_all_video`)
2. Завантажує поточну версію з `videos_version.json`
3. Порівнює версії та виконує відповідні дії:

**Сценарій 1**: Запис є і версія співпадає
- Дія: Нічого не робиться, залишається неактивний стан (користувач уже переглянув усі поточні відео)

**Сценарій 2**: Запис є і версія відрізняється
- Дія: 
  - Змінює стан thumbnail з неактивного на активний (з'явилися нові відео!)
  - Видаляє поточний запис з `localStorage` (щоб уникнути захаращення)

**Сценарій 3**: Запису немає
- Дія: Змінює стан thumbnail з неактивного на активний (користувач ще не переглядав рілси)

### Призначення системи

Ця система дозволяє:
- Автоматично сповіщати користувачів про нові відео у рілсах через активний значок Trends
- Уникати повторних сповіщень для вже переглянутого контенту
- Підтримувати актуальність стану значка Trends на сайті
- Мінімізувати кількість даних у `localStorage`

### Наше завдання при додаванні нових відео

Для активації значка Trends після додавання нових відео потрібно:

1. **Додати нові відео** до масиву `gamesData` у файлі `src/gameDataSets/current_videos.ts`
2. **Збільшити версію** у файлі `public/videos_version.json` на одиницю (наприклад, з `"17"` на `"18"`)
   - ⚠️ Версія має бути цілим числом
   - ⚠️ Версія має йти по зростанню (не можна використовувати меншу версію)
3. **Виконати деплой** проєкту командою `npm run deploy`

Після деплою користувачі сайту автоматично побачать активний значок Trends, що сигналізує про наявність нових відео.
